# Setting Up the Compiler

Compact has two tools you need to know about:

- **`compact`** — the Compact command-line tool. This is what you install and use day-to-day. It manages toolchain versions and wraps the compiler, formatter, and fixup tool.
- **`compactc`** — the actual compiler. You typically don't invoke this directly; `compact compile` calls it for you.

> Source: [docs.midnight.network/getting-started/installation](https://docs.midnight.network/getting-started/installation), [docs.midnight.network/compact/compilation-and-tooling/compiler-usage](https://docs.midnight.network/compact/compilation-and-tooling/compiler-usage), [docs.midnight.network/compact/compilation-and-tooling/dev-tool-usage](https://docs.midnight.network/compact/compilation-and-tooling/dev-tool-usage)

---

## Prerequisites

Before you start, make sure you have:

- **Google Chrome** browser
- **Visual Studio Code**
- **Docker Desktop** — [docker.com/products/docker-desktop](https://www.docker.com/products/docker-desktop/)
- **Lace Wallet** (Chrome extension) — needed to interact with the Midnight network

> Development is supported on **Linux and Mac**. Windows is not supported at this time.

---

## Step 1: Install the `compact` CLI

Run the installer script:

```bash
curl --proto '=https' --tlsv1.2 -LsSf https://github.com/midnightntwrk/compact/releases/latest/download/compact-installer.sh | sh
```

The installer automatically adds `compact` to your shell's `PATH`. Reload your shell config for the change to take effect:

```bash
source ~/.zshrc   # for zsh
# or
source ~/.bashrc  # for bash
```

If `compact` is still not found after restarting, add it manually:

```bash
export PATH="$HOME/.compact/bin:$PATH"
```

---

## Step 2: Install the Compiler

Once the CLI is installed, pull down the latest compiler version:

```bash
compact update
```

This downloads the latest Compact toolchain and sets it as the default.

### Verify the installation

```bash
compact --version         # version of the compact CLI
compact compile --version # version of the compiler
which compact             # installation path
```

---

## Step 3: Start the Proof Server

The proof server generates zero-knowledge proofs locally for transactions. It runs as a Docker container. Make sure Docker Desktop is running first, then:

```bash
docker run -p 6300:6300 midnightntwrk/proof-server:8.0.3 midnight-proof-server -v
```

This occupies the terminal while running. The server listens at `http://localhost:6300`.

> To use the local proof server with the Lace Midnight wallet, go to **Settings → Midnight** and select `Local (http://localhost:6300)`.

---

## Step 4: Install the VS Code Extension

1. Download the [VSIX package](https://raw.githubusercontent.com/midnight-ntwrk/releases/gh-pages/artifacts/vscode-extension/compact-0.2.13/compact-0.2.13.vsix)
2. In VS Code, go to **Extensions → Install from VSIX** and select the downloaded file

This gives you syntax highlighting and code snippet completion for `.compact` files.

## Compiling a Contract

The basic compile command takes a source file and a target directory:

```bash
compact compile src/counter.compact obj/counter
```

This produces:

```
obj/counter/contract/index.d.ts        # TypeScript type definitions
obj/counter/contract/index.js          # compiled JavaScript
obj/counter/contract/index.js.map      # source map
obj/counter/contract/contract-info.json

obj/counter/zkir/foo.zkir              # ZK circuit (one per exported circuit that touches the ledger)

obj/counter/keys/foo.prover            # proving key
obj/counter/keys/foo.verifier          # verifier key
```

Only circuits that touch the public ledger get zkir and key files. Pure circuits that don't read or write ledger state do not.

### Skip ZK Key Generation

Generating proving keys is slow. During development when you only care about TypeScript output:

```bash
compact compile --skip-zk src/counter.compact obj/counter
```

### Compile with a Specific Toolchain Version

```bash
compact compile +0.30.0 src/counter.compact obj/counter
```

The version must be fully qualified (`+0.30.0`, not `+0.30`). If that version isn't installed, run `compact update 0.30.0` first.

## Other Useful Commands

### Formatting

```bash
compact format src/          # format all .compact files in a directory
compact format --check src/  # check formatting without modifying (good for CI)
```

### Fixup

`fixup` applies source-level transformations when the language changes between versions — beyond just formatting:

```bash
compact fixup src/           # apply fixup to all files
compact fixup --check src/   # check what would change without modifying
```

### Manage Versions

```bash
compact list               # available versions on remote
compact list --installed   # locally installed versions
compact check              # check if an update is available
```

## The Standard Library

Most contracts need the standard library. Add this at the top of your `.compact` file:

```compact
import CompactStandardLibrary;
```

`CompactStandardLibrary` is built into the compiler — it is not a file on disk.

## Troubleshooting

**`compact: command not found`** — the binary is not in your PATH. Add `export PATH="$HOME/.compact/bin:$PATH"` to your shell config and reload.

**Docker connection errors** — Docker Desktop is not running. Open it and wait for the engine to start before running the proof server.

**Port 6300 already in use** — run the proof server on a different port:

```bash
docker run -p 6301:6300 midnightntwrk/proof-server:8.0.3 midnight-proof-server -v
```

Then update your app config to point to the new port.
