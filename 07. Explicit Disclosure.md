# Explicit Disclosure

Explicit disclosure is Compact's mechanism for controlling what private data flows into the public world. It is enforced at compile time by a component of the Compact compiler called the **witness protection program**.

> Source: [docs.midnight.network/compact/reference/explicit-disclosure](https://docs.midnight.network/compact/reference/explicit-disclosure)

---

## The Core Rule

A Compact program must explicitly declare its intention to disclose data that might be private before:

1. Storing it in the public ledger
2. Returning it from an exported circuit
3. Passing it to another contract

This makes **privacy the default** and **disclosure an explicit exception**. The compiler rejects any program that violates this rule.

---

## What Counts as Witness Data

Witness data originates from:

- Return values of `witness` function calls
- Arguments passed to exported circuits
- Arguments passed to the contract constructor

Any value **derived** from witness data is also considered witness data — arithmetic, type conversions, struct construction, function calls. The taint follows the data everywhere.

---

## `disclose()` — The Explicit Declaration

`disclose(expr)` tells the compiler: *"I know this expression may contain witness data, and I am intentionally disclosing it."*

It has **no runtime effect**. It does not encrypt, anonymize, or transform the value. It is purely a compile-time annotation — a signed waiver that you are making a deliberate choice.

```compact
witness getBalance(): Bytes<32>;
export ledger balance: Bytes<32>;

// Without disclose() — compiler error
export circuit record(): [] {
  balance = getBalance();
}

// With disclose() — compiles
export circuit record(): [] {
  balance = disclose(getBalance());
}
```

---

## The Compiler Error

When `disclose()` is missing, the compiler aborts with a detailed error showing the witness source, the path the data took, and the disclosure point:

```
Exception: line 6 char 11:
  potential witness-value disclosure must be declared but is not:
    witness value potentially disclosed:
      the return value of witness getBalance at line 2 char 1
    nature of the disclosure:
      ledger operation might disclose the witness value
    via this path through the program:
      the right-hand side of = at line 6 char 11
```

Read the path from top to bottom to trace exactly how witness data traveled to the disclosure point.

---

## Indirect Disclosure — The Compiler Always Catches It

Witness data cannot be hidden from the compiler by passing it through arithmetic, structs, or helper circuits:

```compact
circuit obfuscate(x: Field): Field {
  return x + 73;  // output is still witness data
}

export circuit record(): [] {
  const s = S { x: getBalance() as Field };
  const x = obfuscate(s.x);
  balance = x as Bytes<32>;  // compiler error — path traced through obfuscate
}
```

The compiler's abstract interpreter follows the witness taint through every operation:

```
the binding of s at line 11 char 3
the argument to obfuscate at line 12 char 13
the computation at line 7 char 10
the binding of x at line 12 char 3
the right-hand side of = at line 13 char 11
```

Passing witness data through a function does not reset the taint. The `disclose()` can be placed **anywhere along the path** — on the witness call, inside the helper circuit, or at the final disclosure point.

---

## Disclosure Via Return Values

Returning witness-derived data from an exported circuit also counts as disclosure — even if nothing goes to the ledger:

```compact
witness getBalance(): Uint<64>;

// compiler error: return value discloses witness data
export circuit balanceExceeds(n: Uint<64>): Boolean {
  return getBalance() > n;
}

// correct
export circuit balanceExceeds(n: Uint<64>): Boolean {
  return disclose(getBalance()) > n;
}
```

Even a comparison result — a `Boolean` derived from witness data — counts as a disclosure and must be declared.

---

## Where to Place `disclose()`

**General rule:** place `disclose()` as close to the disclosure point as possible.

This minimizes the scope of what you are declaring disclosed. If the value travels along multiple paths, placing the wrapper too early risks accidentally authorizing disclosure on paths you didn't intend.

**Exception — structured values:** for tuples, vectors, or structs, wrap only the portions that contain witness data:

```compact
// if only the second field is private, disclose only that
const result = [publicValue, disclose(privateValue)];
```

Wrapping the entire struct would declare all fields as disclosed, including fields that were never witness-tainted.

**Exception — always-safe witnesses:** if a witness always returns data that is already safe to disclose (e.g., it returns a public hash), wrapping the call itself is fine:

```compact
const pk = disclose(computePublicKey());
```

## Standard Library Exceptions

The compiler makes two special-case decisions for standard library functions:

| Function | Treated as witness data? | Why |
|----------|--------------------------|-----|
| `transientCommit(e)` | **No** | The random nonce sufficiently hides the input |
| `transientHash(e)` | **Yes** | A bare hash may not hide the input sufficiently |

This means you can store a `transientCommit` result in the ledger without a `disclose()` wrapper — the commitment itself is not considered a disclosure of the underlying value.

```compact
// no disclose() needed — the commitment hides the value
ledger commitment: Field;

export circuit commit(v: Field): [] {
  const nonce = freshNonce();
  commitment = transientCommit(v, nonce);  // compiles without disclose()
}
```

But a bare hash requires `disclose()`:

```compact
ledger hashed: Field;

export circuit storeHash(v: Field): [] {
  hashed = disclose(transientHash(v));  // disclose() required
}
```

---

## How the Witness Protection Program Works

The compiler implements a static abstract interpreter. Instead of computing actual runtime values, it tracks whether each value **contains witness data**. When it encounters an undeclared disclosure point — a ledger write, an exported return, or a cross-contract call — it halts and reports the error with the full taint path.

`disclose(e)` tells the abstract interpreter to treat `e` as if it contains no witness data from that point forward, regardless of whether it actually does.

---

## Auditing Your Contract's Privacy Model

Every `disclose()` in your codebase is a record of a conscious decision. You can audit your contract's privacy model by searching for `disclose()` — every hit is a place where witness data crosses into the public world.

If you find a `disclose()` you didn't intend, the fix is architectural, not cosmetic.

---

## Quick Reference

| Scenario | Requires `disclose()`? |
|----------|----------------------|
| Store witness value in ledger | Yes |
| Return witness value from exported circuit | Yes |
| Return comparison result involving witness data | Yes |
| Pass witness value to another contract | Yes |
| Use witness value inside a circuit (no disclosure) | No |
| Store `transientCommit(witnessValue)` in ledger | No |
| Store `transientHash(witnessValue)` in ledger | Yes |
| Pass witness value to a non-exported circuit | No |
