# Data Types

Compact is statically and strongly typed. Every expression has a type known at compile time. All types have fixed sizes at compile time.

> Source: [Compact Types](https://docs.midnight.network/compact/reference/compact-reference#compact-types)

## Primitive Types

### Boolean

```compact
const flag: Boolean = true;
const other = false;
```

### Field

The set of unsigned integers up to the order of the native prime field. Values cannot be compared with `<`, `<=`, `>`, `>=`, only equality.

```compact
const f: Field = 42;
const g = 0xdeadbeef as Field;  // large literals must be cast
```

### Uint<n>: Sized Integer

```compact
const x: Uint<8> = 255;
const y: Uint<64> = 1000000;
```

### Uint<0..n>: Bounded Integer

```compact
const age: Uint<0..150> = 25;   // values 0 to 149
const idx: Uint<0..256> = 100;  // values 0 to 255
```

`Uint<0..n>` is a subtype of `Uint<0..m>` if `n ≤ m`.

### Bytes<n>

A byte vector of exactly `n` bytes. Used for hashing, keys, identifiers.

```compact
const key: Bytes<32> = pad(32, "midnight:example:key");
const hash: Bytes<32> = persistentHash<Field>(42);
```

### Opaque<"string"> and Opaque<"Uint8Array">

Allows foreign JavaScript data to be stored without being inspected by Compact code. Opaque values are transparent in TypeScript but opaque to circuits, circuits see only their hash.

```compact
witness getMessage(): Opaque<"string">;
export ledger message: Opaque<"string">;

export circuit post(): [] {
  message = disclose(getMessage());
}
```

**Important:** opaque values are not hidden on-chain.

## Composite Types

### Tuples [T1, T2, ..., Tn]

```compact
const pair: [Field, Boolean] = [42, true];
const first = pair[0];   // Field
const second = pair[1];  // Boolean
```

### Vector<n, T>

Homogeneous fixed-length sequence.

```compact
const v: Vector<4, Uint<8>> = [1, 2, 3, 4];
const w = [10, 20, 30];  // inferred as Vector<3, ...>
```

Vectors support `map` and `fold`.

## Program-Defined Types

### struct

Named collection of fields. Typing is **nominal**, two structs are the same type only if they have the same name.

```compact
struct Point {
  x: Uint<32>,
  y: Uint<32>,
}

const p = Point { x: 10, y: 20 };
const xVal = p.x;
```

Structs cannot be recursive.

### enum

A named set of variants. The first element is the default value.

```compact
enum Direction { up, down, left, right }

const d = Direction.up;
assert(s == State.SET, "Expected SET state");
```

## Type Aliases

### Structural alias

```compact
type Pair<T> = [T, T];
type Hash = Bytes<32>;
```

Fully interchangeable.

### Nominal alias

```compact
new type UserId = Bytes<32>;
new type TokenAmount = Uint<64>;
```

Distinct types. Requires explicit cast between them:

```compact
const id: UserId = someBytes as UserId;
const raw: Bytes<32> = id as Bytes<32>;
```

## Subtyping Rules

- Any type `T` is a subtype of itself
- `Uint<0..n>` is a subtype of `Uint<0..m>` if `n ≤ m`
- `Uint<0..n>` is a subtype of `Field` if `n-1` is within the field range
- `[T1, ..., Tn]` is a subtype of `[S1, ..., Sn]` if each `Ti` is a subtype of `Si`

## Type Casting

```compact
const x: Uint<64> = 42;
const y = x as Field;          // widen to Field
const z = x as Uint<0..1000>;  // narrow — dynamic error if out of range
const b = someBytes as UserId; // nominal alias cast
```

## Default Values

Every type has a default value, used when initializing ledger fields:

| Type | Default |
|------|---------|
| `Boolean` | `false` |
| `Uint<n>`, `Uint<0..n>`, `Field` | `0` |
| `Bytes<n>` | `n` zero bytes |
| `[T1, ..., Tn]` | tuple of defaults |
| `Vector<n, T>` | vector of defaults |
| `struct` | each field to default |
| `enum` | first variant |
| `Opaque<"string">` | zero-length string |
| `Opaque<"Uint8Array">` | zero-length array |

```compact
const empty = default<Bytes<32>>;
const zero = default<Uint<64>>;
```
