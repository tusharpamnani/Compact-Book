# Data Types

Compact is statically and strongly typed. Every expression has a type known at compile time, and the compiler rejects any program that doesn't type-check. All types have sizes fixed at compile time — this is a hard requirement of ZK circuit compilation.

> Source: [docs.midnight.network/compact/reference/compact-reference#compact-types](https://docs.midnight.network/compact/reference/compact-reference#compact-types), [docs.midnight.network/compact/data-types/opaque_data](https://docs.midnight.network/compact/data-types/opaque_data)

---

## Primitive Types

### `Boolean`

The type of `true` and `false`. Only two possible values.

```compact
const flag: Boolean = true;
const other = false;
```

### `Field`

The set of unsigned integers up to the order of the native prime field of the ZK proving system. This is the largest unsigned integer type in Compact. Field values cannot be compared with `<`, `<=`, `>`, `>=` — only equality (`==`, `!=`).

```compact
const f: Field = 42;
const g = 0xdeadbeef as Field;  // large literals must be cast to Field
```

### `Uint<n>` — Sized Integer

A sized unsigned integer using up to `n` bits. `Uint<32>` is equivalent to `Uint<0..4294967296>`.

```compact
const x: Uint<8> = 255;
const y: Uint<64> = 1000000;
const z: Uint<32> = 0;
```

### `Uint<0..n>` — Bounded Integer

A bounded unsigned integer in the range `[0, n)`. More explicit than sized integers. These are what the compiler works with internally — sized integers are syntactic sugar for bounded ones.

```compact
const age: Uint<0..150> = 25;   // values 0 to 149
const idx: Uint<0..256> = 100;  // values 0 to 255
```

**Subtyping:** `Uint<0..n>` is a subtype of `Uint<0..m>` if `n ≤ m`. This means a narrower type can be used where a wider type is expected without an explicit cast.

### `Bytes<n>`

A byte vector of exactly `n` bytes. Used for hashing, keys, and identifiers. String literals in Compact have type `Bytes<n>` where `n` is the length of the UTF-8 encoding.

```compact
const key: Bytes<32> = pad(32, "midnight:example:key");
const hash: Bytes<32> = persistentHash<Field>(42);
```

`pad(n, s)` pads string `s` with zero bytes to reach length `n`. The string must not be longer than `n`.

### `Opaque<"string">` and `Opaque<"Uint8Array">`

Opaque types allow "foreign" JavaScript data to be stored and passed around by a DApp without being inspected by Compact code. Opaque values are transparent in TypeScript but opaque to circuits — circuits see only their hash.

Currently only two tags are supported: `"string"` and `"Uint8Array"`.

```compact
witness getMessage(): Opaque<"string">;
export ledger message: Opaque<"string">;

export circuit post(): [] {
  message = disclose(getMessage());
}
```

**Important:** opaque values are not hidden on-chain. `Uint8Array` is represented as an array of bytes, and `string` as its UTF-8 encoding. The "opaque" refers to Compact's inability to inspect the structure — not to on-chain privacy.

## Composite Types

### Tuples `[T1, T2, ..., Tn]`

Heterogeneous fixed-length sequences. Each element can have a different type.

```compact
const pair: [Field, Boolean] = [42, true];
const triple: [Uint<8>, Uint<16>, Uint<32>] = [1, 2, 3];
```

Access by index:

```compact
const first = pair[0];   // Field
const second = pair[1];  // Boolean
```

Two tuple types are different if they have different lengths or if any element type differs.

### `Vector<n, T>`

A homogeneous fixed-length sequence — shorthand for a tuple where all elements have the same type. `Vector<3, Field>` is exactly the same type as `[Field, Field, Field]`.

```compact
const v: Vector<4, Uint<8>> = [1, 2, 3, 4];
const w = [10, 20, 30];  // inferred as Vector<3, ...>
```

Vectors support `map` and `fold`. Use `...` to spread one vector into another:

```compact
const combined = [...v, ...w];
```

## Program-Defined Types

### `struct`

Named collection of fields. Typing is **nominal** — two structs are the same type only if they have the same name and same fields in the same order.

```compact
struct Point {
  x: Uint<32>,
  y: Uint<32>,
}

struct Pair<T> {
  first: T;
  second: T;
}
```

Create a struct value:

```compact
const p = Point { x: 10, y: 20 };
const q = Point { x: p.x, y: 0 };
```

Access fields:

```compact
const xVal = p.x;
```

Destructure in a pattern:

```compact
circuit sumPoint({x, y}: Point): Uint<64> {
  return x + y;
}
```

Structs cannot be recursive — a struct cannot contain a field of its own type, directly or indirectly.

### `enum`

A named set of variants. The first element is the default value.

```compact
enum Direction { up, down, left, right }
enum State { UNSET, SET }
```

Reference a variant:

```compact
const d: Direction = Direction.up;
const s: State = State.SET;
```

Compare:

```compact
assert(s == State.SET, "Expected SET state");
```

Two enum types are the same if they have the same name and same elements in the same order.

## Type Aliases

### Structural alias (interchangeable)

```compact
type Pair<T> = [T, T];
type Hash = Bytes<32>;
```

`Hash` and `Bytes<32>` are the same type — fully interchangeable.

### Nominal alias (distinct type)

```compact
new type UserId = Bytes<32>;
new type TokenAmount = Uint<64>;
```

`UserId` and `Bytes<32>` are compatible (same representation, can be cast between them) but are **not** the same type. You cannot pass a `Bytes<32>` where a `UserId` is expected without an explicit cast.

```compact
const id: UserId = someBytes as UserId;
const raw: Bytes<32> = id as Bytes<32>;
```

Nominal types are useful for preventing mixing up values of the same underlying type that have different semantics.

## Subtyping Rules

Compact has a limited but precise subtyping system:

- Any type `T` is a subtype of itself
- `Uint<0..n>` is a subtype of `Uint<0..m>` if `n ≤ m`
- `Uint<0..n>` is a subtype of `Field` if `n-1` is within the field range
- `[T1, ..., Tn]` is a subtype of `[S1, ..., Sn]` if each `Ti` is a subtype of `Si`
- `Vector<n, T>` is a subtype of `[S1, ..., Sn]` if `T` is a subtype of each `Si`

Subtypes can be used where supertypes are expected without an explicit cast.

## Type Casting

Use `as` to cast between compatible types:

```compact
const x: Uint<64> = 42;
const y = x as Field;          // widen to Field
const z = x as Uint<0..1000>;  // narrow — dynamic error if value out of range
const b = someBytes as UserId; // nominal alias cast
```

Narrowing casts can cause dynamic errors at runtime if the value doesn't fit in the target type.

## Default Values

Every type has a default value, used when initializing ledger fields:

| Type | Default |
|------|---------|
| `Boolean` | `false` |
| `Uint<n>`, `Uint<0..n>` | `0` |
| `Field` | `0` |
| `Bytes<n>` | `n` zero bytes |
| `[T1, ..., Tn]` | tuple of defaults |
| `Vector<n, T>` | vector of `n` defaults |
| `struct` | each field set to its default |
| `enum` | first declared variant |
| `Opaque<"string">` | zero-length string |
| `Opaque<"Uint8Array">` | zero-length array |

Obtain the default value explicitly:

```compact
const empty = default<Bytes<32>>;
const zero = default<Uint<64>>;
const origin = default<Point>;
```

## Quick Reference

| Type | Description | Example |
|------|-------------|---------|
| `Boolean` | True or false | `true` |
| `Field` | Full ZK field integer | `42 as Field` |
| `Uint<n>` | n-bit unsigned integer | `Uint<64>` |
| `Uint<0..n>` | Bounded unsigned integer | `Uint<0..256>` |
| `Bytes<n>` | n-byte vector | `Bytes<32>` |
| `Opaque<"string">` | Foreign string (opaque to circuits) | — |
| `Opaque<"Uint8Array">` | Foreign byte array (opaque to circuits) | — |
| `[T1, T2]` | Heterogeneous tuple | `[Field, Boolean]` |
| `Vector<n, T>` | Homogeneous fixed vector | `Vector<4, Uint<8>>` |
| `struct` | Named field collection | `struct Point { x, y }` |
| `enum` | Named variants | `enum State { UNSET, SET }` |
| `type X = T` | Structural alias | fully interchangeable |
| `new type X = T` | Nominal alias | distinct, requires cast |
