# Modules and Imports

Compact uses a static module system for namespace management and splitting programs across files.

> Source: [Modules, Exports, and Imports](https://docs.midnight.network/compact/reference/compact-reference#modules-exports-and-imports)

## Defining a Module

```compact
module Math {
  export circuit add(a: Field, b: Field): Field {
    return a + b;
  }

  export circuit mul(a: Field, b: Field): Field {
    return a * b;
  }

  circuit helper(x: Field): Field {  // not exported
    return x + 1;
  }
}
```

Bindings inside a module are invisible outside unless explicitly exported.

## Exporting from a Module

```compact
module M {
  export struct Point { x: Field, y: Field }
  export circuit distance(p: Point): Field { ... }
}

// or separate export
module M {
  struct Point { x: Field, y: Field }
  circuit distance(p: Point): Field { ... }
  export { Point, distance };
}
```

## Importing

```compact
import Math;              // everything
import { add } from Math; // specific
import Math prefix Math$; // with prefix
import { add as plus } from Math;  // renamed
```

## Generic Modules

```compact
module Container<T, #N> {
  export circuit first(v: Vector<N, T>): T { return v[0]; }
  export circuit last(v: Vector<N, T>): T { return v[N - 1]; }
}

import Container<Field, 4>;
```

Generic modules must be specialized at import.

## Importing from Files

```compact
import MyModule;
// looks for MyModule.compact in the same directory

import "utils/Math";
// looks for utils/Math.compact relative to importing file
```

Rules:
- File must contain exactly one module definition
- If not found, it is a static error
- Search path is set via `--compact-path` or `COMPACT_PATH`

## Include Files

`include` splices contents directly into the current file:

```compact
include "shared/types.compact";
```

Useful for sharing type definitions.

## Top-Level Exports

```compact
export circuit transfer(to: Bytes<32>, amount: Uint<64>): [] { ... }
export struct TokenInfo { name: Bytes<32>, supply: Uint<64> }
export ledger totalSupply: Uint<64>;
```

Rules:
- No two exported circuits at top level can share the same name
- Generic circuits cannot be exported — they must be specialized first

## The Standard Library

The standard library is built-in, not a file:

```compact
import CompactStandardLibrary;
```

## Module Order

- A module must be **defined before** any import of that module
- Program-defined types used as generic arguments must be **defined before** the import

## Practical Example

**`state.compact`**

```compact
module State {
  enum STATUS { UNSET, SET }
  ledger status: STATUS;
  ledger value: Field;

  export circuit init(v: Field): [] {
    value = disclose(v);
    status = STATUS.SET;
  }

  export circuit get(): Field {
    assert(status == STATUS.SET, "Not initialized");
    return value;
  }
}
```

**`main.compact`**

```compact
pragma language_version 0.22;
import CompactStandardLibrary;
import State;

constructor(v: Field) {
  init(v);
}

export circuit getValue(): Field {
  return get();
}
```

## Quick Reference

| Syntax | Effect |
|--------|--------|
| `module M { ... }` | Define module `M` |
| `export circuit f(...)` | Export from module/top-level |
| `export { f, g }` | Export multiple |
| `import M` | Import all exports of `M` |
| `import { f } from M` | Import specific |
| `import M prefix P$` | Import with prefix |
| `import { f as g } from M` | Rename |
| `import M<T, 4>` | Generic module |
| `import "path/M"` | Import from file path |
| `include "file.compact"` | Splice file inline |
| `import CompactStandardLibrary` | Built-in stdlib |
