# Witnesses

A witness is a callback function that the TypeScript dApp provides at runtime. Witnesses are how circuits access private data, data that lives on the user's local machine and never touches the chain.

> Source: [Declaring Witnesses for Private State Management](https://docs.midnight.network/compact/reference/compact-reference#declaring-witnesses-for-private-state-management)

## The Mental Model

The name *witness* comes from zero-knowledge literature. In a ZK proof, a witness is the evidence needed to believe a statement is true. In Compact, it is the evidence the contract needs to believe an operation is permissible.

## Witness + Circuit Flow

![Witness + Circuit Flow](images/witness_circuit_flow.png)

**Key points:**
- Witness runs locally on user's device: never exposed
- Circuit compiles to ZK proof: proves correct execution
- Only the proof goes on-chain: inputs stay private

## Declaring a Witness

```compact
witness secretKey(): Bytes<32>;
witness getBalance(addr: Bytes<32>): Uint<64>;
witness userNonce(): Field;
witness getItem<T>(index: Uint<32>): T;  // generic
```

Witness declarations have no body, the body is provided by the TypeScript dApp.

## Calling a Witness

```compact
export circuit clear(): [] {
  const sk = secretKey();           // call witness: private input
  const pk = publicKey(round, sk); // compute: still private
  assert(authority == pk, "Not authorized");
  state = State.UNSET;
  round.increment(1);
}
```

The witness call happens locally on the user's machine. The ZK proof is generated proving the circuit executed correctly without revealing `sk`.

## Witness Data is Tracked

The compiler tracks witness data through every operation, arithmetic, type conversions, struct construction, function calls.

```compact
export circuit example(): [] {
  const s = getSecret();
  const doubled = s + s;       // still witness data
  const converted = s as Uint<64>; // still witness data
  ledger = doubled;            // compiler error: undeclared disclosure
}
```

## Witness Data and `disclose()`

When witness data needs to flow into the public ledger, you must wrap it in `disclose()`:

```compact
// Without: compiler error
export circuit record(): [] {
  balance = getBalance();
}

// With: compiles
export circuit record(): [] {
  balance = disclose(getBalance());
}
```

`disclose()` does not encrypt. It is a compile-time annotation saying "I know this is going public, intentionally."

## The Error Message

When you forget `disclose()`, the compiler tells you exactly where the witness data came from and how it traveled:

```
Exception: line 6 char 11:
  potential witness-value disclosure must be declared but is not:
    witness value potentially disclosed:
      the return value of witness getBalance at line 2 char 1
    nature of the disclosure:
      ledger operation might disclose the witness value
```

## Indirect Disclosure

The compiler catches disclosure even when witness data travels through helper circuits:

```compact
circuit obfuscate(x: Field): Field {
  return x + 73;  // output is still witness data
}

export circuit record(): [] {
  const s = getBalance() as Field;
  const x = obfuscate(s);
  balance = x as Bytes<32>;  // compiler still catches this
}
```

## Best Practice

Place `disclose()` **as close to the disclosure point as possible**. This minimizes the scope of what you are declaring disclosed.

## Standard Library Exceptions

| Function | Witness-tainted? | Why |
|----------|------------------|-----|
| `transientCommit(e)` | **No** | Random nonce provides sufficient hiding |
| `transientHash(e)` | Yes | Bare hash may not hide input |

This means `transientCommit` can commit private values for on-chain storage without `disclose()` on the commitment itself.

## Critical: Witness Results Are Untrusted

> Do not assume in your contract that the code of any `witness` function is the code that you wrote. Any DApp may provide any implementation it wants. Results should be treated as untrusted input.

The ZK proof guarantees the circuit's *logic* ran correctly given whatever inputs witnesses returned; it does not guarantee witnesses returned sensible values. Your contract must verify witness outputs with `assert`.

## Witness vs Circuit

| | `witness` | `circuit` |
|--|-----------|-----------|
| Body | Implemented in TypeScript | Implemented in Compact |
| Execution | Runs locally | Compiled to ZK circuit |
| Output | Private data | Proven result |
| Trust | Untrusted, must validate | Enforced correct by ZK |
