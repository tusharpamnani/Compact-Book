# Ledger ADTs

Ledger ADTs (Abstract Data Types) are the collection types available for on-chain state in Compact. They go beyond simple scalar values — `Map`, `Set`, `MerkleTree`, `List`, and `Counter` all have specific operations and semantics designed for on-chain use.

> Source: [docs.midnight.network/compact/reference/ledger-adt](https://docs.midnight.network/compact/reference/ledger-adt)

---

## How Ledger Operations Work

Each ledger field with an ADT type supports a set of operations invoked as:

```compact
fieldName.operation(args)
```

For example:

```compact
import CompactStandardLibrary;
ledger cnt: Counter;

export circuit inc(): [] {
  cnt.increment(1);
  // or shorthand:
  cnt += 1;
}
```

Some ADTs have shorthand syntax for common operations (read → field reference, write → `=`, increment → `+=`, decrement → `-=`).

---

## Counter

A simple unsigned counter. The key advantage over a plain `Cell<Uint<64>>` is that `Counter` does not read the current value when incrementing or decrementing — so the transaction is not committed to a specific current value and is less likely to be rejected due to contention.

```compact
import CompactStandardLibrary;
ledger votes: Counter;

export circuit vote(): [] {
  votes += 1;              // increment by 1
}

export circuit retract(): [] {
  votes -= 1;              // decrement by 1
}

export circuit getVotes(): Uint<64> {
  return votes;            // read
}
```

### Operations

| Operation | Syntax | Description |
|-----------|--------|-------------|
| `increment(n)` | `field += n` | Increment by `n` (Uint<16>) |
| `decrement(n)` | `field -= n` | Decrement by `n` — errors if below 0 |
| `read()` | `field` | Read current value as `Uint<64>` |
| `lessThan(n)` | — | Returns `Boolean` |
| `resetToDefault()` | — | Reset to 0 |

## Cell (Plain Types)

When you declare `ledger f: T` for any ordinary Compact type `T`, it is implicitly a `Cell<T>`. Programmers cannot write `Cell` explicitly.

```compact
ledger owner: Bytes<32>;

export circuit setOwner(addr: Bytes<32>): [] {
  owner = disclose(addr);      // shorthand for owner.write(...)
}

export circuit getOwner(): Bytes<32> {
  return owner;                // shorthand for owner.read()
}
```

### Operations

| Operation | Syntax | Description |
|-----------|--------|-------------|
| `read()` | `field` | Read current value |
| `write(v)` | `field = v` | Overwrite the value |
| `resetToDefault()` | — | Reset to type's default value |

## Set

An unbounded set of unique values of type `T`. Inserting a duplicate is a no-op.

```compact
import CompactStandardLibrary;
ledger allowlist: Set<Bytes<32>>;

export circuit addAddress(addr: Bytes<32>): [] {
  allowlist.insert(disclose(addr));
}

export circuit removeAddress(addr: Bytes<32>): [] {
  allowlist.remove(disclose(addr));
}

export circuit isAllowed(addr: Bytes<32>): Boolean {
  return allowlist.member(addr);
}
```

### Operations

| Operation | Description |
|-----------|-------------|
| `insert(elem)` | Add element |
| `remove(elem)` | Remove element |
| `member(elem)` | Returns `Boolean` — is element present? |
| `isEmpty()` | Returns `Boolean` |
| `size()` | Returns `Uint<64>` — number of entries |
| `resetToDefault()` | Clear the set |
| `[Symbol.iterator]()` | TypeScript only — iterate over elements |

## Map

An unbounded key-value mapping. Keys must be ordinary Compact types. Values can be ordinary Compact types **or** other ledger-state types (except `Kernel`) — enabling nested ADTs.

```compact
import CompactStandardLibrary;
ledger balances: Map<Bytes<32>, Uint<64>>;

export circuit setBalance(addr: Bytes<32>, amount: Uint<64>): [] {
  balances.insert(disclose(addr), disclose(amount));
}

export circuit getBalance(addr: Bytes<32>): Uint<64> {
  return balances.lookup(addr);
}

export circuit hasBalance(addr: Bytes<32>): Boolean {
  return balances.member(addr);
}

export circuit removeBalance(addr: Bytes<32>): [] {
  balances.remove(disclose(addr));
}
```

### Operations

| Operation | Description |
|-----------|-------------|
| `insert(key, value)` | Insert or overwrite key-value pair |
| `insertDefault(key)` | Insert key with the value type's default |
| `lookup(key)` | Returns value — the returned value may itself be an ADT |
| `member(key)` | Returns `Boolean` |
| `remove(key)` | Remove a key |
| `isEmpty()` | Returns `Boolean` |
| `size()` | Returns `Uint<64>` |
| `resetToDefault()` | Clear the map |
| `[Symbol.iterator]()` | TypeScript only — iterate over `[key, value]` pairs |

### Nested Maps

Map values can themselves be ledger-state types, enabling nested structures:

```compact
import CompactStandardLibrary;
ledger fld: Map<Boolean, Map<Field, Counter>>;

export circuit initInner(b: Boolean): [] {
  fld.insert(disclose(b), default<Map<Field, Counter>>);
}

export circuit initCounter(b: Boolean, n: Field): [] {
  fld.lookup(b).insert(disclose(n), default<Counter>);
}

export circuit increment(b: Boolean, n: Field, k: Uint<16>): [] {
  fld.lookup(b).lookup(n) += disclose(k);
}

export circuit read(b: Boolean, n: Field): Uint<64> {
  return fld.lookup(b).lookup(n);
}
```

**Rules for nested maps:**
- Nested ledger-state values must be initialized before first use — accessing an uninitialized nested value is a dynamic error
- The entire indirection chain must be used in a single expression — you cannot save a partial lookup in a `const` and use it later
- Only `Map` values can contain other ledger-state types — nesting elsewhere is a static error
- `Kernel` cannot be nested inside a `Map`

## List

An unbounded ordered list. Elements are pushed and popped from the front.

```compact
import CompactStandardLibrary;
ledger queue: List<Field>;

export circuit enqueue(v: Field): [] {
  queue.pushFront(disclose(v));
}

export circuit dequeue(): [] {
  queue.popFront();
}

export circuit peek(): Field {
  const h = queue.head();
  // head() returns Maybe<Field> — handle accordingly in TypeScript
  return h.value;
}
```

### Operations

| Operation | Description |
|-----------|-------------|
| `pushFront(value)` | Add to front of list |
| `popFront()` | Remove first element |
| `head()` | Returns `Maybe<value_type>` — safe on empty list |
| `isEmpty()` | Returns `Boolean` |
| `length()` | Returns `Uint<64>` |
| `resetToDefault()` | Clear the list |
| `[Symbol.iterator]()` | TypeScript only — iterate over elements |

## MerkleTree

A bounded Merkle tree of depth `n` where `2 ≤ n ≤ 32`, containing values of type `T`. Use when you need to prove membership against the **current** root.

```compact
import CompactStandardLibrary;
ledger members: MerkleTree<8, Bytes<32>>;

export circuit addMember(leaf: Bytes<32>): [] {
  members.insert(disclose(leaf));
}

export circuit checkMembership(root: MerkleTreeDigest): Boolean {
  return members.checkRoot(root);
}
```

### Operations (Compact)

| Operation | Description |
|-----------|-------------|
| `insert(item)` | Insert at first free index |
| `insertHash(hash)` | Insert a pre-hashed `Bytes<32>` leaf |
| `insertIndex(item, index)` | Insert at specific index |
| `insertHashIndex(hash, index)` | Insert pre-hashed leaf at specific index |
| `insertIndexDefault(index)` | Insert default value — emulates removal |
| `checkRoot(rt)` | Returns `Boolean` — is `rt` the current root? |
| `isFull()` | Returns `Boolean` |
| `resetToDefault()` | Clear the tree |

### Operations (TypeScript only)

| Operation | Description |
|-----------|-------------|
| `root()` | Get current `MerkleTreeDigest` |
| `firstFree()` | Get first free index as `bigint` |
| `pathForLeaf(index, leaf)` | Get `MerkleTreePath` for a known index/leaf pair |
| `findPathForLeaf(leaf)` | Search O(n) for a leaf — avoid on large trees |

## HistoricMerkleTree

Like `MerkleTree` but retains past roots. Use when you need to prove membership against **any past root** — for example, in nullifier-based systems where a user proves they were in a set at a previous point in time.

```compact
import CompactStandardLibrary;
ledger commitments: HistoricMerkleTree<16, Bytes<32>>;

export circuit addCommitment(leaf: Bytes<32>): [] {
  commitments.insert(disclose(leaf));
}

// Accepts any past root, not just the current one
export circuit verifyOldMembership(root: MerkleTreeDigest): Boolean {
  return commitments.checkRoot(root);
}
```

Everything in `MerkleTree` also applies to `HistoricMerkleTree`, with these additions:

| Operation | Description |
|-----------|-------------|
| `checkRoot(rt)` | Returns `Boolean` — is `rt` **any past** root? |
| `resetHistory()` | Compact — discard history, keep only current root valid |
| `history()` | TypeScript only — iterate over all valid past roots |

## Kernel

A special ledger-state type that provides access to built-in ledger operations not tied to a specific field. Declared as a ledger field but used to query and affect global chain state.

```compact
import CompactStandardLibrary;
ledger kern: Kernel;

export circuit checkTime(deadline: Uint<64>): Boolean {
  return kern.blockTimeLessThan(deadline);
}

export circuit selfAddress(): ContractAddress {
  return kern.self();
}
```

### Selected Kernel Operations

| Operation | Description |
|-----------|-------------|
| `self()` | Returns the contract's own `ContractAddress` |
| `blockTimeLessThan(t)` | Is current block time < `t` (Unix seconds)? |
| `blockTimeGreaterThan(t)` | Is current block time > `t`? |
| `balance(tokenType)` | Current unshielded token balance |
| `mintShielded(domainSep, amount)` | Mint shielded tokens |
| `mintUnshielded(domainSep, amount)` | Mint unshielded tokens |
| `claimUnshieldedCoinSpend(...)` | Authorize an unshielded transfer |
| `checkpoint()` | Mark all preceding execution as atomic |

`Kernel` cannot be nested inside a `Map` or any other ledger-state type.

## Choosing the Right ADT

| Use case | ADT |
|----------|-----|
| Single mutable value | `ledger f: T` (Cell) |
| Monotonically growing counter with low contention | `Counter` |
| Membership tracking | `Set<T>` |
| Per-key storage (balances, allowances) | `Map<K, V>` |
| Per-key nested storage | `Map<K, Map<K2, V>>` |
| Ordered queue | `List<T>` |
| ZK membership proofs (current state) | `MerkleTree<n, T>` |
| ZK membership proofs (historical state) | `HistoricMerkleTree<n, T>` |
| Block time, token operations, contract address | `Kernel` |
