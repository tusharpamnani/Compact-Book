# Standard Library

The Compact standard library is built into the compiler — not a file on disk. Import it at the top of any contract that needs it:

```compact
import CompactStandardLibrary;
```

> Source: [docs.midnight.network/compact/standard-library](https://docs.midnight.network/compact/standard-library), [docs.midnight.network/compact/standard-library/exports](https://docs.midnight.network/compact/standard-library/exports)

---

## Common Types

### `Maybe<T>`

An optional value. If `isSome` is `false`, `value` should be `default<T>` by convention.

```compact
struct Maybe<T> {
  isSome: Boolean;
  value: T;
}
```

Used by `List.head()`, `MerkleTree.findPathForLeaf()`, and anywhere a value may or may not be present.

```compact
circuit safeHead(list: List<Field>): Field {
  const h = list.head();
  assert(h.isSome, "List is empty");
  return h.value;
}
```

Construct with `some` and `none`:

```compact
circuit some<T>(value: T): Maybe<T>;
circuit none<T>(): Maybe<T>;
```

---

### `Either<A, B>`

A disjoint union. If `isLeft` is `true`, `left` is populated; otherwise `right`. The other should be `default<>` by convention.

```compact
struct Either<A, B> {
  isLeft: Boolean;
  left: A;
  right: B;
}
```

Used heavily in token operations to specify recipients as either a `ContractAddress` or `UserAddress`:

```compact
// send to a user
sendUnshielded(color, amount, right<ContractAddress, UserAddress>(userAddr));

// send to a contract (including self)
sendUnshielded(color, amount, left<ContractAddress, UserAddress>(kernel.self()));
```

Construct with `left` and `right`:

```compact
circuit left<A, B>(value: A): Either<A, B>;
circuit right<A, B>(value: B): Either<A, B>;
```

---

### `ContractAddress`

The address of a contract, as 32 bytes.

```compact
struct ContractAddress { bytes: Bytes<32>; }
```

Get the current contract's own address via `Kernel.self()`.

---

### `UserAddress`

The public key of a user, as 32 bytes.

```compact
struct UserAddress { bytes: Bytes<32>; }
```

---

### `ZswapCoinPublicKey`

The public key used to send shielded coins to a user.

```compact
struct ZswapCoinPublicKey { bytes: Bytes<32>; }
```

---

### `MerkleTreeDigest`

A Merkle tree root, represented as a single `Field`.

```compact
struct MerkleTreeDigest { field: Field; }
```

---

### `MerkleTreePath<#n, T>`

A path in a depth-`n` Merkle tree leading to a leaf of type `T`. Used with `merkleTreePathRoot` to verify membership:

```compact
struct MerkleTreePath<#n, T> {
  leaf: T;
  path: Vector<n, MerkleTreePathEntry>;
}
```

---

### `ShieldedCoinInfo`

Describes a newly created shielded coin. The `nonce` should be unique — use `evolveNonce` to derive it deterministically.

```compact
struct ShieldedCoinInfo {
  nonce: Bytes<32>;
  color: Bytes<32>;
  value: Uint<128>;
}
```

---

### `QualifiedShieldedCoinInfo`

Describes an existing shielded coin on the ledger, ready to be spent. Includes the Merkle tree index.

```compact
struct QualifiedShieldedCoinInfo {
  nonce: Bytes<32>;
  color: Bytes<32>;
  value: Uint<128>;
  mtIndex: Uint<64>;
}
```

## Hashing and Commitment Functions

These are the most commonly used standard library functions for privacy patterns.

### `transientHash<T>(value: T): Field`

A circuit-efficient hash from any value to a `Field`. **Not guaranteed to persist between compiler upgrades** — do not store the result in ledger state. Use for consistency checks only.

Output is still treated as witness data by the compiler — `disclose()` is required if the result is stored in the ledger or returned from an exported circuit.

```compact
const h = transientHash<[Field, Field]>([a, b]);
```

---

### `transientCommit<T>(value: T, rand: Field): Field`

A circuit-efficient commitment. **Not guaranteed to persist between upgrades** — use for temporary consistency checks, not ledger storage. Unlike `transientHash`, the output is **not** treated as witness data by the compiler (because the nonce hides the input). No `disclose()` required even if stored in the ledger.

```compact
const commit = transientCommit<Field>(secret, nonce);
```

---

### `persistentHash<T>(value: T): Bytes<32>`

A persistent SHA-256 hash. **Guaranteed to persist between upgrades.** Use this for values stored in ledger state — keys, identities, authority hashes.

Output is still treated as witness data — `disclose()` required at disclosure points.

```compact
circuit publicKey(round: Field, sk: Bytes<32>): Bytes<32> {
  return persistentHash<Vector<3, Bytes<32>>>(
    [pad(32, "midnight:lock:pk"), round as Bytes<32>, sk]
  );
}
```

---

### `persistentCommit<T>(value: T, rand: Bytes<32>): Bytes<32>`

A persistent SHA-256 commitment. **Guaranteed to persist between upgrades.** Use this for commitments stored in ledger state. Like `transientCommit`, the output is not treated as witness data.

```compact
const commitment = persistentCommit<Uint<64>>(balance, nonce);
```

---

### Choosing Between Transient and Persistent

| Function | Output type | Persists across upgrades | Use for ledger storage? | Witness-tainted output? |
|----------|-------------|--------------------------|-------------------------|------------------------|
| `transientHash` | `Field` | No | No | Yes |
| `transientCommit` | `Field` | No | No | No |
| `persistentHash` | `Bytes<32>` | Yes | Yes | Yes |
| `persistentCommit` | `Bytes<32>` | Yes | Yes | No |

---

### `degradeToTransient(x: Bytes<32>): Field`

Converts a `persistentHash`/`persistentCommit` output (`Bytes<32>`) to a `Field` for use in `transientHash`/`transientCommit`.

### `upgradeFromTransient(x: Field): Bytes<32>`

Converts a `Field` back to `Bytes<32>`, inverse of `degradeToTransient`.

## Merkle Tree Functions

### `merkleTreePathRoot<#n, T>(path: MerkleTreePath<n, T>): MerkleTreeDigest`

Computes the Merkle root from a path. Use inside a circuit to verify membership without storing all leaves:

```compact
witness getMembershipPath(): MerkleTreePath<8, Bytes<32>>;

export circuit verifyMembership(): [] {
  const path = getMembershipPath();
  const root = merkleTreePathRoot<8, Bytes<32>>(path);
  assert(members.checkRoot(root), "Not a member");
}
```

### `merkleTreePathRootNoLeafHash<#n>(path: MerkleTreePath<n, Bytes<32>>): MerkleTreeDigest`

Like `merkleTreePathRoot` but assumes leaves are already hashed externally.

## Elliptic Curve Functions

### `ecMulGenerator(b: Field): NativePoint`

Multiplies the primary group generator of the embedded curve by scalar `b`. Used to derive public keys from scalars.

### `ecMul(a: NativePoint, b: Field): NativePoint`

Multiplies a point by a scalar.

### `ecAdd(a: NativePoint, b: NativePoint): NativePoint`

Adds two elliptic curve points.

### `hashToCurve<T>(value: T): NativePoint`

Maps an arbitrary value to a curve point with unknown discrete logarithm. Useful for Pedersen-style commitments.

## Block Time Functions

All block time values are Unix timestamps in seconds.

```compact
circuit blockTimeLt(time: Uint<64>): Boolean;   // current time < time
circuit blockTimeGte(time: Uint<64>): Boolean;  // current time >= time
circuit blockTimeGt(time: Uint<64>): Boolean;   // current time > time
circuit blockTimeLte(time: Uint<64>): Boolean;  // current time <= time
```

Example — enforce a deadline:

```compact
export circuit claimBefore(deadline: Uint<64>): [] {
  assert(blockTimeLt(deadline), "Deadline has passed");
  // ...
}
```

## Token Functions

### `nativeToken(): Bytes<32>`

Returns the token type identifier for the native tDUST token.

### `tokenType(domainSep: Bytes<32>, contract: ContractAddress): Bytes<32>`

Derives a globally unique token type for a given contract and domain separator. Contracts can only mint tokens for their own domain separators.

### `ownPublicKey(): ZswapCoinPublicKey`

Returns the `ZswapCoinPublicKey` of the user creating the transaction. Useful for identifying the caller in circuits.

### Unshielded Token Operations

```compact
// mint new unshielded tokens to a recipient
mintUnshieldedToken(domainSep, value, recipient): Bytes<32>

// send unshielded tokens
sendUnshielded(color, amount, recipient): []

// receive unshielded tokens (required to accept incoming unshielded transfers)
receiveUnshielded(color, amount): []
```

Balance checks (prefer these over `unshieldedBalance` to avoid contention):

```compact
unshieldedBalanceLt(color, amount): Boolean
unshieldedBalanceGte(color, amount): Boolean
unshieldedBalanceGt(color, amount): Boolean
unshieldedBalanceLte(color, amount): Boolean
```

> **Note:** `unshieldedBalance` returns the balance at the start of execution, not updated during the transaction. Using it commits the transaction to an exact balance value — prefer the comparison functions instead.

### Shielded Token Operations

```compact
// mint a new shielded coin to a recipient
mintShieldedToken(domainSep, value, nonce, recipient): ShieldedCoinInfo

// receive a shielded coin (coin must be an output in the transaction)
receiveShielded(coin: ShieldedCoinInfo): []

// send from a coin owned by the contract to a recipient
sendShielded(input, recipient, value): ShieldedSendResult

// send from a coin created in the current transaction
sendImmediateShielded(input, target, value): ShieldedSendResult

// burn address — any coin sent here is destroyed
shieldedBurnAddress(): Either<ZswapCoinPublicKey, ContractAddress>

// deterministically derive a nonce
evolveNonce(index, nonce): Bytes<32>
```
