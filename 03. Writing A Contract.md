# Writing a Contract

A Compact contract is built from four kinds of pieces: a `pragma`, `ledger` declarations, `circuit` definitions, and `witness` declarations. This note walks through building a complete contract from scratch, explaining what each piece does and why.

> Source: [docs.midnight.network/compact/reference/writing](https://docs.midnight.network/compact/reference/writing)

---

## The Contract We're Building

A simple value store with three operations:

- `get` — read the current value (anyone can call this)
- `set` — write a value (only when unset)
- `clear` — clear the value (only the user who last called `set` can do this)

---

## Step 1: The Pragma and Imports

Every contract starts with a language version declaration and imports:

```compact
pragma language_version 0.22;

import CompactStandardLibrary;
```

- `pragma language_version` pins the contract to a specific Compact language version. If the compiler doesn't support it, it will error.
- `import CompactStandardLibrary` gives you access to hashing, commitment, and ledger utilities.

---

## Step 2: Define Your Types

Custom types are declared with `enum` or `struct`. Here we need a state enum:

```compact
enum State {
  UNSET,
  SET
}
```

`State.UNSET` and `State.SET` are the two possible values. The first element (`UNSET`) is also the default value of this type.

---

## Step 3: The Ledger Section

The `ledger` section declares what the contract stores on-chain. Each `ledger` declaration is one field of the contract's public state:

```compact
export ledger authority: Bytes<32>;

export ledger value: Uint<64>;

export ledger state: State;

export ledger round: Counter;
```

- `authority` — a hash of the secret key of whoever last called `set`. Used to authorize `clear`.
- `value` — the stored 64-bit unsigned integer.
- `state` — whether the value is currently set or unset.
- `round` — a counter used to prevent linkability between rounds (explained below).

All ledger fields are initialized to their default values on deployment. A `constructor` can override those defaults:

```compact
constructor(sk: Bytes<32>, v: Uint<64>) {
  authority = disclose(publicKey(round, sk));
  value = disclose(v);
  state = State.SET;
}
```

Note the `disclose()` wrapper — any private value flowing into public ledger state must be explicitly disclosed. The compiler enforces this.

---

## Step 4: Helper Circuits

A `circuit` in Compact is like a function, but it compiles to a ZK circuit. Here's a private helper that computes a "public key" by hashing a round number and a secret key:

```compact
circuit publicKey(round: Field, sk: Bytes<32>): Bytes<32> {
  return persistentHash<Vector<3, Bytes<32>>>(
           [pad(32, "midnight:examples:lock:pk"), round as Bytes<32>, sk]);
}
```

This is not exported — it's only used internally by other circuits. The `persistentHash` function comes from `CompactStandardLibrary` and produces a `Bytes<32>` suitable for storing in ledger state.

---

## Step 5: Exported Circuits (Entry Points)

Exported circuits are the entry points of the contract — what users call in transactions.

### `get` — unrestricted read

```compact
export circuit get(): Uint<64> {
  assert(state == State.SET, "Attempted to get uninitialized value");
  return value;
}
```

`assert` acts as a guard. If the condition is false, the circuit halts with the given error message.

### `set` — write a value

```compact
export circuit set(v: Uint<64>): [] {
  assert(state == State.UNSET, "Attempted to set initialized value");
  const sk = secretKey();
  const pk = publicKey(round, sk);
  authority = disclose(pk);
  value = disclose(v);
  state = State.SET;
}
```

`secretKey()` is a witness call — it retrieves private data from the user's local machine (explained next). The return type `[]` means this circuit returns nothing (unit type).

### `clear` — authorized clear

```compact
export circuit clear(): [] {
  assert(state == State.SET, "Attempted to clear uninitialized value");
  const sk = secretKey();
  const pk = publicKey(round, sk);
  assert(authority == pk, "Attempted to clear without authorization");
  state = State.UNSET;
  round.increment(1);
}
```

This proves (via ZK) that the caller knows the secret key that hashes to the stored `authority`, without ever revealing the secret key itself.

---

## Step 6: Witnesses

A `witness` is a callback function that the TypeScript DApp provides at runtime. It supplies private data to circuits without exposing it on-chain:

```compact
witness secretKey(): Bytes<32>;
```

The body is not written in Compact — it is implemented in TypeScript by the DApp. Witness results must be treated as untrusted input: any DApp could provide any implementation.

---

## What Is and Isn't Confidential

Two rules govern confidentiality in this model:

- **Confidential:** all data that is not a ledger field and is not an argument or return value of a ledger operation
- **Enforced correct:** all computation that is not done in a `witness` function

In `clear`, the `secretKey()` output stays confidential. What gets proven is that *its hash* matches `authority`. The ZK proof guarantees this check happened correctly without revealing the secret.

### Why the `round` Counter?

Without it, the `pk` value would be the same every time the same user calls `set`. This would allow linking different rounds to the same user (a privacy leak). By including `round` in the hash, each round produces a different `pk` — breaking that linkability.

## The Full Contract

```compact
pragma language_version 0.22;

import CompactStandardLibrary;

enum State {
  UNSET,
  SET
}

export ledger authority: Bytes<32>;

export ledger value: Uint<64>;

export ledger state: State;

export ledger round: Counter;

constructor(sk: Bytes<32>, v: Uint<64>) {
  authority = disclose(publicKey(round, sk));
  value = disclose(v);
  state = State.SET;
}

circuit publicKey(round: Field, sk: Bytes<32>): Bytes<32> {
  return persistentHash<Vector<3, Bytes<32>>>(
           [pad(32, "midnight:examples:lock:pk"), round as Bytes<32>, sk]);
}

export circuit get(): Uint<64> {
  assert(state == State.SET, "Attempted to get uninitialized value");
  return value;
}

witness secretKey(): Bytes<32>;

export circuit set(v: Uint<64>): [] {
  assert(state == State.UNSET, "Attempted to set initialized value");
  const sk = secretKey();
  const pk = publicKey(round, sk);
  authority = disclose(pk);
  value = disclose(v);
  state = State.SET;
}

export circuit clear(): [] {
  assert(state == State.SET, "Attempted to clear uninitialized value");
  const sk = secretKey();
  const pk = publicKey(round, sk);
  assert(authority == pk, "Attempted to clear without authorization");
  state = State.UNSET;
  round.increment(1);
}
```

## Structure Summary

| Element | Purpose |
|---------|---------|
| `pragma language_version` | Pin to a specific language version |
| `import CompactStandardLibrary` | Bring in hashing, commitment, and ledger utilities |
| `enum` / `struct` | Define custom types |
| `ledger` | Declare on-chain public state fields |
| `constructor` | Initialize state on deployment |
| `circuit` (unexported) | Private helper functions |
| `export circuit` | Entry points — callable in transactions |
| `witness` | Declare callbacks implemented by the TypeScript DApp |
| `disclose()` | Required wrapper when private data flows into public state |
| `assert()` | Runtime guard — halts the circuit if the condition is false |
