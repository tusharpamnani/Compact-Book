# Circuits

A circuit is the basic operational unit of a Compact contract. It corresponds closely to a function in most languages but is designed to compile into a zero-knowledge circuit. That design constraint shapes everything about how circuits work.

> Source: [docs.midnight.network/compact/reference/compact-reference#circuit-definitions](https://docs.midnight.network/compact/reference/compact-reference#circuit-definitions)

---

## What Makes a Circuit Different from a Function

- **No recursion.** Circuits cannot call themselves, directly or indirectly. This is a hard requirement — ZK circuits must be finite.
- **No unbounded loops.** All `for` loops must have bounds known at compile time.
- **All types have fixed sizes at compile time.** You cannot have dynamically-sized structures.
- **Variables are immutable.** Once bound via `const`, a variable's value cannot change within that lifetime.

These aren't limitations to work around. They are what makes a circuit compilable into a ZK proof.

---

## Syntax

```compact
export pure circuit name<GenericParams>(param: Type, ...): ReturnType {
  // body
}
```

- `export` — makes this circuit an entry point callable from TypeScript
- `pure` — optional; asserts the circuit has no side effects (no ledger reads/writes, no witness calls)
- `circuit` — the keyword
- generic parameters — optional, enclosed in `<>`
- parameters — each must have an explicit type annotation
- return type — must be explicitly declared for named circuits

---

## A Simple Circuit

```compact
circuit add(a: Uint<64>, b: Uint<64>): Uint<64> {
  return a + b;
}
```

This is a private helper circuit — not exported. Only other circuits in the same contract can call it.

---

## An Exported Circuit

```compact
export circuit get(): Uint<64> {
  assert(state == State.SET, "Value is not set");
  return value;
}
```

`export` makes this callable as a transaction entry point. The compiler enforces that no two exported circuits at the top level share the same name.

---

## Pure vs Impure Circuits

A circuit is **pure** if it computes outputs from inputs only — no ledger reads or writes, no witness calls.

A circuit is **impure** if it:
- reads or writes any ledger field
- calls a witness
- calls another impure circuit

The `pure` modifier is optional but useful — it tells the compiler to verify your claim and error if the circuit is actually impure:

```compact
pure circuit hashPair(a: Field, b: Field): Field {
  return transientHash<[Field, Field]>([a, b]);
}
```

If `hashPair` accidentally called a witness, the compiler would catch it.

Pure circuits appear in the `PureCircuits` TypeScript type exported by the compiler, which lets you call them without generating a proof.

## Parameters and Destructuring

Parameters are patterns, not just identifiers. You can destructure tuples and structs directly in the parameter list:

```compact
// simple
circuit double(x: Field): Field {
  return x + x;
}

// tuple destructuring
circuit sumPair([a, b]: [Field, Field]): Field {
  return a + b;
}

// struct destructuring
struct Point { x: Uint<32>, y: Uint<32> }

circuit sumPoint({x, y}: Point): Uint<64> {
  return x + y;
}

// rename a field during destructuring
circuit useX({x: val, y}: Point): Uint<32> {
  return val;
}
```

## Return Types

Every named circuit must declare its return type explicitly. Use `[]` for circuits that return nothing (unit type):

```compact
export circuit clear(): [] {
  state = State.UNSET;
}
```

A circuit body that falls off the end without a `return` statement is treated as `return;` — equivalent to returning `[]`.

---

## `const` Bindings

Local variables in circuits are declared with `const`. They are immutable:

```compact
circuit compute(x: Field): Field {
  const doubled = x + x;
  const result = doubled * 3;
  return result;
}
```

You can bind multiple variables in one `const` statement:

```compact
const a = x[0], b = x[1];
```

You can also destructure:

```compact
const [a, b] = pair;
const {x, y} = point;
```

---

## `assert`

`assert` is the primary guard mechanism. It halts circuit execution with an error message if the condition is false:

```compact
assert(state == State.SET, "Value is not initialized");
assert(balance >= amount, "Insufficient balance");
```

Both arguments are required — the condition and the error string.

---

## `if` Statements

```compact
if (condition) {
  // then
}

if (condition) {
  // then
} else {
  // else
}
```

The condition must be `Boolean`. One-armed `if` cannot be the "then" branch of a two-armed `if`.

---

## `for` Loops

Compact supports two forms of `for`, both bounded at compile time:

### Iterate over a vector or tuple

```compact
circuit sumVec(v: Vector<4, Field>): Field {
  const result = 0;  // note: use fold for accumulation — see below
  for (const x of v) {
    // x is each element
  }
  return result;
}
```

### Iterate over a numeric range

```compact
for (const i of 0..10) {
  // i goes 0, 1, 2, ..., 9
}
```

Both `start` and `end` must be compile-time constants or generic natural-number parameters. `return` cannot be used inside a `for` body.

---

## `map` and `fold`

For accumulation and transformation over vectors, use `map` and `fold` instead of `for`:

```compact
// double every element
circuit doubleAll(v: Vector<4, Field>): Vector<4, Field> {
  return map((x) => x + x, v);
}

// sum all elements
circuit sumAll(v: Vector<4, Field>): Field {
  return fold((acc, x) => acc + x, 0, v);
}
```

These are the idiomatic way to accumulate values in Compact because `for` loops cannot use `return`.

---

## Generic Circuits

Circuits can have generic type and size parameters:

```compact
circuit identity<T>(x: T): T {
  return x;
}

circuit firstOf<#N, T>(v: Vector<N, T>): T {
  return v[0];
}
```

Generic circuits must be specialized at the call site:

```compact
const x = identity<Field>(42);
const head = firstOf<4, Field>([1, 2, 3, 4]);
```

Generic circuits cannot be exported from the top level — they must be specialized before export.

---

## Anonymous Circuits (Lambdas)

Compact supports anonymous circuits (used with `map`, `fold`, etc.):

```compact
map((x) => x + 1, v)          // expression body
map((x) => { return x + 1; }, v)  // block body
```

Anonymous circuits can have type annotations but don't require them.

---

## Shadowing

A `const` binding in an inner block can shadow an outer binding:

```compact
circuit example(): Field {
  const answer = 42;
  {
    const answer = 12;
    assert(answer != 42, "inner answer is 12");
  }
  return answer; // returns 42
}
```

## Quick Reference

| Feature | Compact |
|---------|---------|
| Define a circuit | `circuit name(p: T): R { ... }` |
| Export as entry point | `export circuit name(...)` |
| Declare pure | `pure circuit name(...)` |
| Local variable | `const x = expr;` |
| Guard | `assert(cond, "msg")` |
| Conditional | `if (cond) { } else { }` |
| Bounded loop | `for (const i of 0..N) { }` |
| Iterate over vector | `for (const x of vec) { }` |
| Transform vector | `map(fn, vec)` |
| Accumulate | `fold(fn, init, vec)` |
| Return nothing | `: []` |
| No recursion | — compiler enforces this |
