# Circuits

A circuit is the basic operational unit of a Compact contract. It compiles into a zero-knowledge circuit.

> Sources: [Circuit Definitions](https://docs.midnight.network/compact/reference/compact-reference#circuit-definitions)

## What Makes a Circuit Different

- **No recursion.** Circuits cannot call themselves
- **No unbounded loops.** All `for` loops have bounds known at compile time
- **All types have fixed sizes** at compile time
- **Variables are immutable.** Once bound via `const`, a value cannot change

These aren't limitations; they are what makes a circuit compilable into a ZK proof.

## Syntax

```compact
export pure circuit name<GenericParams>(param: Type, ...): ReturnType {
  // body
}
```

- `export`: makes this an entry point callable from TypeScript
- `pure`: optional; asserts the circuit has no side effects
- `circuit`: the keyword
- generic parameters: optional, enclosed in `<>`
- parameters: each must have explicit type annotation
- return type: must be explicitly declared

## Simple Circuits

```compact
circuit add(a: Uint<64>, b: Uint<64>): Uint<64> {
  return a + b;
}

export circuit get(): Uint<64> {
  assert(state == State.SET, "Value is not set");
  return value;
}
```

## Pure vs Impure

A circuit is **pure** if it computes outputs from inputs only, no ledger reads/writes, no witness calls.

```compact
pure circuit hashPair(a: Field, b: Field): Field {
  return transientHash<[Field, Field]>([a, b]);
}
```

If `hashPair` accidentally called a witness, the compiler would catch it. Pure circuits appear in the `PureCircuits` TypeScript type.

## Parameters and Destructuring

```compact
circuit double(x: Field): Field { return x + x; }

// tuple destructuring
circuit sumPair([a, b]: [Field, Field]): Field { return a + b; }

// struct destructuring
struct Point { x: Uint<32>, y: Uint<32> }
circuit sumPoint({x, y}: Point): Uint<64> { return x + y; }

// rename a field
circuit useX({x: val}: Point): Uint<32> { return val; }
```

## Return Types

Use `[]` for circuits that return nothing:

```compact
export circuit clear(): [] {
  state = State.UNSET;
}
```

## Local Bindings

```compact
circuit compute(x: Field): Field {
  const doubled = x + x;
  const result = doubled * 3;
  return result;
}

const [a, b] = pair;
const {x, y} = point;
```

## Control Flow

```compact
assert(state == State.SET, "Value is not initialized");

if (condition) {
  // then
} else {
  // else
}
```

## For Loops

Both forms are bounded at compile time:

```compact
// iterate over vector
for (const x of v) { }

// iterate over numeric range (0..N excludes N)
for (const i of 0..10) { }
```

`return` cannot be used inside a `for` body.

## map and fold

For accumulation and transformation:

```compact
circuit doubleAll(v: Vector<4, Field>): Vector<4, Field> {
  return map((x) => x + x, v);
}

circuit sumAll(v: Vector<4, Field>): Field {
  return fold((acc, x) => acc + x, 0, v);
}
```

These are idiomatic for Compact because `for` loops cannot use `return`.

## Generic Circuits

```compact
circuit identity<T>(x: T): T { return x; }

circuit firstOf<#N, T>(v: Vector<N, T>): T { return v[0]; }
```

Generic circuits must be specialized at the call site:

```compact
const x = identity<Field>(42);
const head = firstOf<4, Field>([1, 2, 3, 4]);
```

Generic circuits cannot be exported from the top level.
